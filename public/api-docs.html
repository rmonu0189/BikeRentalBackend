All endpoints serve and consume JSON under /v1. Restricted endpoints expect Authorization: Bearer <access_token>. Use the sidebar to set your target server's Base URL. Under each endpoint, use Try this request to test the endpoints live from this page.

Session: Not signed in

Token Flow & Overview

This API implements a secure passwordless sign-in flow via OTP (One-Time Password) using mobile numbers.

1. Send OTP: Request an OTP code to any phone number. If in development, the OTP will be logged to storage/logs/otp.log, and returned directly in the response if OTP_RETURN_CODE_IN_RESPONSE=true is set in `.env`.

2. Verify OTP: Submit the OTP. If the user doesn't exist yet, they will be registered automatically (you can optionally supply full_name and email). The endpoint returns a short-lived **Access Token** (JWT) and a long-lived **Refresh Token**.

3. Token Rotation: Use the Refresh Token to periodically fetch a new Access Token. Refresh tokens rotate upon every refresh call to secure session boundaries.

Sign in & Session Tester

Perform a passwordless login below. Once verified, the tokens will be saved in your browser's tab session. Other Try this request panels will automatically load and attach your authentication token.

POST /v1/auth/otp/send

Generates an OTP challenge for the mobile number. Rate-limited by IP and phone.

Body Parameters

  • phone (string, required) — normalizeable mobile number.
  • client (string, optional) — set to "mobile" for mobile app registration.
  • device_label, user_agent (string, optional).
Example Request Body 
{ "phone": "+919109322140", "client": "mobile" }
Sample Response — 200 OK 
{
  "ok": true,
  "expires_in": 600,
  "debug_otp_code": "123456"
}

debug_otp_code is only returned when OTP_RETURN_CODE_IN_RESPONSE=true in `.env`.
