BikeRental API Reference

All endpoints serve and consume JSON under /v1.

Token Flow & Overview

This API implements a secure passwordless sign-in flow via OTP (One-Time Password) using mobile numbers.

The authentication system relies on JSON Web Tokens (JWT) for secure, stateless requests. Endpoints requiring authentication expect the token in the headers as follows:

Authorization: Bearer <access_token>

Token Authentication Lifecycle

  1. Send OTP: Call POST /auth/otp/send with a phone number. In development mode, the OTP is logged to storage/logs/otp.log and returned in the payload if configured.
  2. Verify OTP: Submit the OTP to POST /auth/otp/verify. If successful, you will receive a short-lived Access Token and a long-lived Refresh Token.
  3. Token Rotation: Use the Refresh Token on POST /auth/refresh to receive a new Access Token. Refresh tokens rotate upon every refresh call.

Sign in & Session Tester

Perform a passwordless login below. Once verified, the tokens will be saved in your browser's tab session. Other Try this request panels will automatically load and attach your authentication token.

POST /v1/auth/otp/send

Authentication: None
Content-Type: application/json

Generates an OTP challenge and sends it to the specified phone number. Rate-limited by IP (30 requests/hour) and phone number (5 requests/15 mins).

Request Body Parameters

Field Type Required Description
phone string Yes Valid mobile number with country code (e.g. "+919109322140").
client string No Client application type, set to "mobile" to allow auto-signup of users.
device_label string No Human-readable description of device.
user_agent string No Custom user agent string override.

Responses & Status Codes

{
  "ok": true,
  "expires_in": 600,
  "debug_otp_code": "123456"
}

debug_otp_code is only returned when OTP_RETURN_CODE_IN_RESPONSE=true is set in `.env`.

{
  "error": "Too many OTP requests",
  "retry_after": 3600
}
